Аудит появится здесь
Все действия администраторов — создание, блокировка, назначение ролей и RLS — будут записаны в журнал аудита.
Правила администрирования
Создание пользователей Пользователь создаётся в Admin администратором
Назначение ролей Роли назначаются администратором портала
MFA для критичных ролей system_admin, security_admin требуют MFA
Аудит изменений Все изменения доступа аудируются
Удаление пользователей Логическое (archive), не физическое удаление
RLS enforcement RLS применяется на backend/DB/API уровне
Excel и BI Excel export и BI reports учитывают RLS пользователя
Хранение паролей Только безопасный hash (bcrypt/argon2)
🛡️ Локальные пользователи заводятся в Admin. Роли назначаются администратором. SSO/AD может быть подключён дополнительно. Пароли никогда не отображаются в UI и не передаются в открытом виде. Все изменения пользователей/ролей/RLS аудируются. RLS enforced на backend/DB/API.
