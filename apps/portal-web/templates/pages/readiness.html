🎯 Бизнес-демо: что можно показать

Полный в системе сценарий рекламной кампании доступен для демонстрации без физической КСО. Физическая доставка на КСО, фактические показы и сканер — заблокированы до закрытия approval gates.

✅ Что уже готово (система + портал)
✅ Загрузка креативов — PNG/JPEG/MP4, портрет 768×1024
✅ Создание и управление кампаниями — статусы, привязка креативов
✅ Настройка расписания — слоты по дням недели, временные окна
✅ Согласование — maker-checker, approve/reject с комментариями
✅ Публикация в backend — batch lifecycle, manifest generation, в системе publish
✅ Плановая отчётность — занятость эфира, кампании по статусам, CSV export
✅ Dashboard — KPI-сводка, process, pilot readiness, план действий
✅ Безопасность — RBAC, RLS, audit trail, safe projection (без raw UUID, secrets, backend URL)
🎬 Сценарий демонстрации (в системе)
1 🎨 Креатив → 2 📢 Кампания → 3 🗓 Расписание → 4 📝 Согласование → 5 📋 Публикация → 6 📈 Отчёт

Полный цикл рекламной кампании: загрузка креатива → создание кампании → настройка расписания → отправка на согласование → решение → подготовка публикации → генерация manifest → backend публикация → выгрузка плановой отчётности в CSV.

Весь сценарий выполняется в backend без физической КСО. Manifest формируется, но не доставляется на устройство.

📋 Статус публикаций См. approval gates
👉 Следующий шаг после появления сканера
  1. Подключить физический сканер к КСО → выполнить Проверка физического сканера (Разрешение на проверку сканера)
  2. Доставить manifest на физическую КСО → проверить парсинг и отображение (Разрешение на доставку)
  3. Запустить sidecar sync на КСО → убедиться, что PoP-события доходят до backend (Разрешение на синхронизацию)
  4. Выполнить 48h+ long-run с мониторингом — память, диск, Chromium, перезапуски (Разрешение на проверку стабильности)
  5. После закрытия всех P0 — решение GO/NO-GO по пилоту (Разрешение на запуск)
  6. Пилот одного КСО → сбор evidence → решение о расширении fleet

Все approval tokens описаны в runbook. Каждый шаг требует отдельного разрешения, последовательно.

Документы: pilot runbook · fallback/rollback runbook · physical approval gates · GO/NO-GO baseline
