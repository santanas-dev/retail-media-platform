# UKM5 UI Safe Zone Mapping

> **Статус:** 📋 Visual Audit (38.0.4)
>
> Дата: 2026-06-24
> Ревизия: 1
>
> **Назначение:** Определить безопасные зоны интерфейса УКМ5 на экране 768×1024 portrait, где потенциально можно показывать рекламу без перекрытия критичных элементов.
> **НЕ:** изменение кода, установка ПО, перезапуск сервисов, изменение УКМ5.

## 2. Состояния экрана

### Состояние 1: Главный экран / Режим ожидания ✅

Удалось посмотреть через `scrot` (read-only screenshot). УКМ5 находится в idle-состоянии: нет активной продажи, товары не выбраны, чек не открыт.

### Состояния, которые НЕ проверялись

| Состояние | Причина |
|---|---|
| Экран выбора/сканирования товаров | Требует сканирования реального товара → операционный риск |
| Экран корзины/чека | Требует добавления товара → нарушает read-only принцип |
| Экран оплаты | Категорически запрещено — фискальный риск |
| Экран ошибки/предупреждения | Не воспроизводился без реальной операции |

## 4. Candidate Ad Zones

### 4.1 Zone A: Top Strip

| Параметр | Значение |
|---|---|
| Координаты | (0, 0) — (768, 120) |
| Размер | 768 × 120 |
| Когда можно показывать | НИКОГДА — перекрывает header + заголовок + close button |
| Когда скрывать | Неприменимо |
| Перекрывает | C1 (header), C2 (close), C3 (yellow header) |
| Риск для покупателя | Потеря информации о магазине/режиме |
| Риск для кассира | Невозможно закрыть/выйти |
| Пригодность | ❌ REJECTED |

### 4.2 Zone B: Mid-Content Overlay

| Параметр | Значение |
|---|---|
| Координаты | (0, 122) — (768, 358) |
| Размер | 768 × 236 |
| Когда можно показывать | **Только idle** — когда нет активной продажи |
| Когда скрывать | Мгновенно при начале сканирования/выбора |
| Перекрывает | C6 (рабочая зона) |
| Риск для покупателя | Низкий в idle (нет товаров) |
| Риск для кассира | Мешает начать продажу, требует быстрого скрытия |
| Пригодность | 🟡 CONDITIONAL — требует idle-detection + fast hide |

### 4.3 Zone C: Product Grid (BEST CANDIDATE) ✅

| Параметр | Значение |
|---|---|
| Координаты | (0, 400) — (768, 640) |
| Размер | 768 × 240 |
| Когда можно показывать | **Только idle** — когда корзина пуста |
| Когда скрывать | При добавлении первого товара / начале сканирования |
| Перекрывает | Ничего критичного в idle (пустая сетка) |
| Риск для покупателя | Минимальный (в idle-режиме) |
| Риск для кассира | Низкий — зона становится рабочей при начале продажи |
| Пригодность | ✅ **RECOMMENDED** для pilot |

**Обоснование:**
- Самая большая свободная зона в idle (240px × 768px = 184,320 px²)
- Безопасно отделена от критичных кнопок (70px gap до payment button, 40px gap до separator)
- Естественно очищается при начале продажи (товары занимают эту зону)
- Может вместить баннер 768×200 или 600×200 по центру

## 5. Recommended Safe Zone

### Primary: **Zone C — Product Grid (y=400-640)**

```
Координаты:  (0, 400) → (768, 640)
Размер:      768 × 240
Площадь:     184,320 px²
Режим:       только idle
```

**Ad creative specs для этой зоны:**

| Параметр | Значение |
|---|---|
| Ширина | до 768 px |
| Высота | до 200 px (240 доступно минус padding 20+20) |
| Позиция | по центру: x=84, w=600 или полная ширина x=0, w=768 |
| Отступ от payment | 80 px (y=720 - 640 = 80) |
| Формат | PNG, JPEG |
| Макс. размер | 200 KB |

### Secondary (fallback): **Zone D — Bottom Strip (y=840-980)**

```
Координаты:  (0, 840) → (768, 980)
Размер:      768 × 140
Режим:       idle
```

⚠️ Использовать только если Zone C недоступна (например, товары в корзине, но нет оплаты).

## 6. Safety Rules

### Обязательные правила (enforced)

| # | Правило | Реализация |
|---|---|---|
| R1 | **Не показывать поверх оплаты** | Ad скрывается при y > 720 (зона payment button) |
| R2 | **Не показывать поверх отмены** | Red close button всегда видим |
| R3 | **Не показывать поверх помощи** | Header всегда видим (y=0-60) |
| R4 | **Не показывать поверх критичных кнопок** | Blue payment button не перекрывается |
| R5 | **Мгновенное скрытие при busy/payment** | < 500 мс после сигнала state adapter |
| R6 | **Kill-switch** | Отключение player не влияет на УКМ5 |
| R7 | **Только synthetic test** | Без реальных чеков/оплат/фискальных данных |
| R8 | **Без чтения чеков/оплат** | State adapter не читает БД УКМ5, чеки, платежи |

### Правила перехода idle ↔ busy

| Состояние | Ad показывается | Триггер перехода |
|---|---|---|
| Idle (нет касаний > 30с) | ✅ Да | — |
| Busy (активная продажа) | ❌ Нет | State adapter: `state=busy` |
| Payment (нажата оплата) | ❌ Нет | State adapter: `state=payment` |
| Error/предупреждение | ❌ Нет | State adapter: `state=error` |
| Неизвестно | ❌ Нет (безопасный default) | State adapter: `state=unknown` |

## 9. Рекомендация для шага 38.0.5 (Portrait Player Design)

1. **Primary ad zone:** Product Grid (y=400-640, 768×240)
2. **Ad creative resolution:** 768×200 (центрировано)
3. **Режим:** idle-only. Скрытие при `state=busy` или `state=payment`
4. **Idle detection:** через state adapter. Сигнал idle: нет касаний экрана > 30 секунд. Не читать чеки/БД
5. **Механика показа:** X11 overlay-окно поверх зоны y=400-640, без рамок, без фокуса
6. **Скрытие:** xdotool unmap или hide окна при busy/payment. < 500 мс
7. **Kill-switch:** systemd stop `kso-player` возвращает экран УКМ5 без изменений
8. **Тестирование:** только synthetic creative, только на test KSO, в нерабочие часы
