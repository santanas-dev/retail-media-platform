## 2. Почему текущий KSO Player нельзя устанавливать

### 2.1 Геометрия не совпадает

| Проектное | Фактическое |
|---|---|
| Экран 1920×1080 ландшафт | Экран 768×1024 **портрет** |
| Ad zone 1440×1080 | Не помещается (нужно 1440, есть 768) |
| Sidebar 480×1080 | Не помещается |
| Creative resolution 1440×1080 | Креативы нуждаются в рескейле/перегенерации |

### 2.2 УКМ5 монопольно владеет экраном

- Chromium запущен в режиме `--kiosk` на весь экран (768×1024, позиция 0,0)
- Нет оконного менеджера с поддержкой нескольких окон (Openbox, но Chromium занимает весь root)
- Запуск второго Chromium приведёт к конфликту за X-дисплей

### 2.3 Второй Chromium создаст конфликт

- GPU-ресурсы (один X-сервер, один GPU-контекст)
- Конфликт за `--kiosk` режим (только одно окно может быть в kiosk)
- Openbox не управляет окнами Chromium в kiosk-режиме явно

### 2.4 RAM ограничена

- Свободно: 2.0 GB
- Второй Chromium: ~200-300 MB
- Python sidecar + player: ~100-150 MB
- Остаток: ~1.5 GB — впритык с риском OOM при пиковых нагрузках УКМ5

### 2.5 Риск повлиять на кассовый интерфейс

- УКМ5 — production кассовая система
- Любое изменение X-окружения может нарушить отображение кассового интерфейса
- Overlay-окно поверх УКМ5 может перехватывать фокус/события

## 3. Принятое решение

### Основной путь: **UKM5 / DS native integration**

Не устанавливать отдельный KSO Player на test KSO. Вместо этого — интегрироваться через API СуперМаг DS или API УКМ5:

- **Загрузка контента:** через DS API (если доступен on-premise DS)
- **Расписание:** через DS API / scheduler
- **Показ:** через существующий Chromium kiosk УКМ5 (встройка или API)
- **Факт показа (PoP):** запрос через DS API или отдельный сбор
- **Backend:** на выделенном сервере, не на КСО

### Fallback: **Embedded widget / iframe внутри УКМ5**

Только если поставщик УКМ5/DS разрешит и предоставит:
- Возможность встройки внешнего HTML/iframe в кассовый интерфейс
- API для управления зонами показа
- Гарантии, что встройка не нарушит сертификацию ККТ

### Overlay — только лабораторный вариант

- Допустим для экспериментов в изолированной среде
- Требует отдельного письменного согласования с поставщиком УКМ5
- **Не является целевой архитектурой**
- Риски: фокус, перехват событий, нарушение кассового UI

### Sidecar-only — не решает отображение

- Sidecar может синхронизировать manifest/media
- State adapter может читать состояние УКМ5
- Но без плеера нет показа рекламы — не является самостоятельным решением

## 6. Отклонённые варианты

| Вариант | Причина отклонения |
|---|---|
| Установка KSO Player на test KSO | Геометрия 768×1024 ≠ 1920×1080. Второй Chromium невозможен |
| Overlay как целевая архитектура | Риск конфликта с кассовым UI. Не согласован с поставщиком |
| Sidecar-only без плеера | Не решает задачу показа рекламы |
| Развёртывание backend на КСО | Недостаточно RAM. Риск конкуренции с УКМ5 за ресурсы |
