# Inline Style Exceptions — 43.8.1

После очистки осталось **62** inline `style="..."` в production templates.
Все — легитимные исключения, которые невозможно заменить CSS-классами.

## Категории исключений

### 1. Динамические Jinja2-выражения (≈45 шт.)

**Файлы:** `dashboard.html`, `reports.html`

**Пример:**
```html
<div class="dist-segment approved" style="width:{{ c_approved_pct }}%;"></div>
```

**Почему нельзя CSS-классом:**
Значение вычисляется сервером на основе данных БД (количество кампаний в статусе).
CSS-классы статичны; fill-N классы работают только для фиксированных значений.

**Безопасность:**
- В `{{ }}` попадают только целые числа 0–100 (проценты)
- Нет secrets, tokens, URLs, barcodes, UUID
- Валидация на уровне backend: значения в диапазоне [0, 100]

### 2. Условные Jinja2-выражения (≈1 шт.)

**Файл:** `reports.html`

**Пример:**
```html
<span style="{% if airtime.conflict_count > 0 %}color:var(--color-error);{% endif %}">
```

**Почему нельзя CSS-классом:**
Условие вычисляется сервером. Альтернатива — передавать флаг в контекст и использовать `{% if %}class="text-error"{% endif %}`, что сделано для всех остальных условных стилей. Этот единичный случай оставлен как есть (минорный рефакторинг).

### 3. Абсолютное позиционирование (≈2 шт.)

**Файл:** `reports.html`

**Пример:**
```html
<div class="threshold-marker" style="left:50%;">50%</div>
```

**Почему нельзя CSS-классом:**
Позиции вычисляются относительно родителя. CSS-классы `left-*` потребовали бы десятки комбинаций и не покрыли бы все возможные значения.

### 4. Специфичные layout-свойства (≈14 шт.)

**Файлы:** `schedule.html`, `creatives.html`, `campaigns_create.html`, `approvals.html`, `publications.html`, `devices.html`

Оставшиеся свойства, для которых нет utility-классов и которые не оправдывают создание отдельных классов:
- `padding:28px 16px` (единичное использование)
- `width:150px;padding:6px 10px;font-size:13px` (edge-case размер)
- `border-bottom:1px solid var(--color-border-light)` в специфичных контекстах
- Минорные flex-обёртки

**Почему не добавлены классы:**
Создание CSS-класса для 1–2 использований увеличивает размер кода без выигрыша в поддерживаемости.
