# UKM 4 State Source Discovery

> Безопасный discovery‑пакет для определения источника состояния
> СуперМаг УКМ 4 на Linux КСО.
>
> **Важно:** этот документ описывает подход к безопасному обнаружению.
> Реальная интеграция с УКМ 4 — отдельный будущий шаг.

## Контекст

КСО (касса самообслуживания):
- Linux, СуперМаг УКМ 4
- Chromium kiosk‑mode
- Экран 1920×1080: реклама 1440×1080 слева, УКМ 4 480×1080 справа

State adapter должен получать состояние УКМ 4:
`idle` → `transaction` → `payment` → `receipt` → `idle`
(+ `unknown`, `error`).

## Что нужно узнать (discovery checklist)

- [ ] Есть ли у УКМ 4 официальный API/SDK для получения состояния?
- [ ] Есть ли файл статуса (JSON/XML/text), который УКМ 4 пишет?
- [ ] Есть ли локальный IPC/endpoint?
- [ ] Можно ли определить состояние по системным признакам без чтения данных УКМ 4?
- [ ] Какой минимальный безопасный способ получения только `idle`/`transaction`/...?

## Запрещено читать

| Категория | Примеры |
|---|---|
| Содержимое чека | товары, SKU, названия, количество |
| Суммы | цена, скидка, итого, НДС |
| Платёжные данные | карты, PAN, токены |
| Персональные данные | телефон, email, customer_id, ФИО |
| Fiscal data | ФН, ФД, ФП, смена, кассир |
| База данных УКМ 4 | таблицы, строки, SQL |
| Логи с ПДн | raw логи с содержимым чеков |
| Скриншоты | экран покупателя/кассира с данными |

## Разрешено собирать (техническая информация)

- Наличие процесса УКМ 4 (имя/pid, без командной строки с секретами)
- Наличие сервиса systemd (имя юнита, статус active/inactive)
- Наличие безопасного status endpoint/file (факт существования, без содержимого)
- Версия приложения (если доступна без ПДн)
- Наличие нужных каталогов по allowlist
- Факт наличия документации/API

## Возможные источники состояния

### 1. Официальный API / SDK УКМ 4

**Что проверить:**
- Документация ServPlus / разработчика УКМ 4
- Наличие REST/gRPC/local-socket API для получения состояния кассы
- Формат ответа: только статус или полные данные

**Разрешено читать:** `status` (idle/transaction/payment/receipt)
**Запрещено читать:** чеки, товары, суммы, карты
**Риски:** API может отдавать больше данных, чем нужно
**Плюсы:** точное состояние из первоисточника
**Минусы:** нужен доступ к документации, может отсутствовать
**Рекомендация:** приоритетный путь, если API существует и безопасен

### 2. Статусный файл УКМ 4

**Что проверить:**
- Пишет ли УКМ 4 файл статуса (например, `/var/lib/supermag/status.json`)
- Формат: только статус или полный слепок
- Частота обновления

**Разрешено читать:** поле `state`/`status`
**Запрещено читать:** содержимое чека внутри того же файла
**Риски:** файл может содержать ПДн в одной структуре со статусом
**Плюсы:** просто, файловая интеграция
**Минусы:** нужно убедиться, что в файле только статус
**Рекомендация:** допустимо, если файл содержит исключительно статус

### 3. Локальный IPC / endpoint

**Что проверить:**
- Unix socket, named pipe, localhost HTTP endpoint от УКМ 4
- Протокол: JSON/Protobuf/plain text

**Разрешено читать:** только статус
**Запрещено читать:** полные данные
**Риски:** endpoint может отдавать больше данных
**Плюсы:** низкая задержка
**Минусы:** зависит от реализации УКМ 4
**Рекомендация:** допустимо при безопасном протоколе

### 4. Системные признаки (без чтения данных УКМ 4)

**Что проверить:**
- Наличие процесса: `ps aux | grep supermag` (только имя)
- Активность окна: X11/Wayland window title
- Режим приложения: kiosk, обслуживание, idle

**Разрешено читать:** имя процесса, window class/title (без customer data)
**Запрещено читать:** аргументы командной строки с секретами, содержимое окна
**Риски:** низкая точность, эвристика
**Плюсы:** не касается данных УКМ 4 вообще
**Минусы:** неточный статус (нельзя отличить transaction от payment)
**Рекомендация:** только как fallback для idle/не-idle

### 5. Отдельный lightweight adapter рядом с УКМ 4

**Что проверить:**
- Возможность установки своего скрипта/сервиса на КСО
- Доступ к безопасному источнику (файл, сокет)

**Разрешено:** читать только статус, писать kso_state.json
**Запрещено:** читать БД/логи/чеки УКМ 4
**Риски:** нужен доступ к КСО для установки
**Плюсы:** полный контроль, fail‑closed
**Минусы:** дополнительный компонент
**Рекомендация:** основной путь, state adapter уже готов к этому

## Discovery script

Скрипт `infra/kso-linux/discovery/ukm4_state_discovery.py`:

```bash
# Только план (безопасно, без сканирования)
python3 infra/kso-linux/discovery/ukm4_state_discovery.py --dry-run

# Проверить процесс по имени
python3 infra/kso-linux/discovery/ukm4_state_discovery.py \
    --process-name-pattern supermag

# Проверить сервис systemd
python3 infra/kso-linux/discovery/ukm4_state_discovery.py \
    --service-name supermag-kiosk

# Проверить статусный файл (только наличие, без чтения содержимого)
python3 infra/kso-linux/discovery/ukm4_state_discovery.py \
    --status-file /var/lib/verny/kso/state/kso_state.json
```

**Скрипт НЕ делает:**
- Рекурсивный обход файловой системы
- grep по логам
- Чтение БД
- Чтение содержимого произвольных файлов
- Сетевые запросы
- Запуск systemctl start/enable/restart

## Текущее состояние

State adapter уже умеет:
- Писать `kso_state.json` атомарно
- Работать в режиме daemon
- Принимать injectable source (`StaticStateSource`, `SequenceStateSource`, ...)

**Не хватает:** реального источника состояния УКМ 4.
Discovery нужен, чтобы безопасно определить этот источник
перед реализацией `KsoUkm4StateSource`.

## Recommended Safe Status-File Contract

Если УКМ 4, официальный API, IPC или отдельный adapter
могут писать безопасный статусный файл — state adapter
уже умеет его читать через `SafeStatusFileSource`.

**Разрешённый формат (JSON):**
```json
{"state": "idle"}
```

**Разрешённые ключи JSON:**
- `state` (обязательный)
- `updated_at_utc`, `source`, `schema_version` (опциональные)

**Разрешённые состояния:** `idle`, `transaction`, `payment`, `receipt`,
`service`, `error`, `maintenance`, `offline`, `unknown`

**Разрешённый формат (plain text):**
```
idle
```
Одно слово, lowercase, из списка разрешённых состояний.

**Разрешённые пути:** `/run/verny/kso/`, `/var/lib/verny/kso/`

**Правила файла:**
- Размер ≤ 1024 байт
- Не лог (не растёт бесконечно)
- Только атомарная запись (tmp → rename)

**Запрещено в файле:**
- Чеки, товары, SKU, суммы, скидки
- Карты, PAN, телефоны, email, customer_id
- Fiscal data, кассир, смена
- Токены, секреты, пароли
- Любые персональные данные

**Пример CLI:**
```bash
python3 -m kso_state_adapter.cli daemon \
    --root /var/lib/verny/kso \
    --source file \
    --source-file /run/verny/kso/ukm4-safe-state.json \
    --health-file /run/verny/kso/state-adapter-health.json
```

## Безопасность

- Все проверки **read‑only**
- Никакие данные УКМ 4 не читаются
- Никакие данные не отправляются наружу
- Output не содержит forbidden-строк
- `--dry-run` — безопасный режим по умолчанию
