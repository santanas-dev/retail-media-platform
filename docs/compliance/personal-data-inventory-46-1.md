# Personal Data Inventory — 46.1

> Retail Media Platform — инвентаризация персональных данных и чувствительной информации
> Версия: 1.0
> Дата: 2026-06-29
> Этап: 46.1 Compliance 152-ФЗ Readiness

## 1. Категории ПДн в системе

### 1.1. Учётные данные пользователей (users)

| Поле | Тип ПДн | Зачем нужно | Где видно | Кто имеет доступ | Минимизация |
|---|---|---|---|---|---|
| `username` | Логин (идентификатор) | Аутентификация, аудит | Admin UI, audit pages | `system_admin`, `security_admin` | Не маскируется (ключевой идентификатор) |
| `email` | Email | Контакт, уведомления | **Не показывается в UI** | Только через backend API | Можно сделать необязательным; в v1 собирается но не отображается |
| `display_name` | Может содержать ФИО | Отображение в UI | Admin UI, devices, campaign create | Все роли (для display) | Не маскируется — служебное имя |
| `password_hash` | Хеш (не ПДн) | Аутентификация | Нигде | Никогда не выдаётся | bcrypt, необратимый |

### 1.2. Журнал входа (login_audit_events)

| Поле | Тип ПДн | Хранение | Кто имеет доступ | Минимизация |
|---|---|---|---|---|
| `username` | Идентификатор | Открытый текст | `security_admin` через API | Оправдано для аудита |
| `ip_hash` | Хеш IP-адреса | SHA-256/IP-хеш | `security_admin` | **Уже хеширован** — сырой IP не хранится |
| `user_agent_hash` | Хеш User-Agent | SHA-256/UA-хеш | `security_admin` | **Уже хеширован** — сырой UA не хранится |

### 1.3. Журнал администрирования (admin_audit_events)

| Поле | Тип ПДн | Содержимое | Кто имеет доступ | Минимизация |
|---|---|---|---|---|
| `actor_user_id` | UUID (reference) | Идентификатор администратора | `security_admin` | UUID, не раскрывает личность напрямую |
| `target_ref` | Идентификатор | username/role code | `security_admin` | Оправдано для аудита |
| `details_json` | Структурированные данные | Без secrets/tokens/passwords | `security_admin` | **Политика safe projection** |

### 1.4. Бизнес-данные (advertisers, campaigns, approvals)

| Поле | Риск ПДн | Где видно | Рекомендация |
|---|---|---|---|
| `advertisers.contacts_json` | **Высокий** — может содержать email, телефон | Admin UI | ⚠️ Маскировать при отображении; добавить предупреждение |
| `advertisers.comment` | Средний | Admin UI | Добавить предупреждение «Не вводить ПДн» |
| `campaigns.comment` | Средний | Campaign pages | Добавить предупреждение «Не вводить ПДн» |
| `approval_requests.comment` | Средний | Approvals page | Добавить предупреждение «Не вводить ПДн» |
| `advertisers.inn` | Низкий (юрлицо) | Admin UI | ИНН юрлица — не персональные данные |
| `advertisers.legal_name` | Низкий (юрлицо) | Admin UI | Наименование юрлица — публичная информация |
| `stores.address` | Низкий (юрлицо) | Stores, inventory | Адрес магазина — не персональные данные |

### 1.5. Данные, которые НЕ собираются

- ❌ Телефонные номера (нет поля phone в users)
- ❌ Паспортные данные
- ❌ Платёжная информация
- ❌ Фискальные данные
- ❌ Данные банковских карт
- ❌ Биометрические данные
- ❌ Специальные категории ПДн
- ❌ Сырые IP-адреса (только хеши)
- ❌ Сырые User-Agent (только хеши)

## 2. Безопасность хранения

### 2.1. Пароли
- Алгоритм: bcrypt
- Хранение: необратимый хеш
- Восстановление: невозможно (только сброс администратором)

### 2.2. Токены
- Access token: JWT (HS256), срок 15 минут
- Refresh token: JWT (HS256), срок 7 дней
- Хранение refresh token: SHA-256 хеш в БД
- Сессия портала: серверный in-memory store; браузерный cookie — opaque session_id

### 2.3. Cookie
- httpOnly: ✅
- SameSite: Lax ✅
- Signed: ✅
- Secure: ❌ (dev без HTTPS; задокументировано для production)
- max_age: 3600 сек (1 час)

## 3. Остаточные риски

| Риск | Уровень | Смягчение |
|---|---|---|
| `contacts_json` без валидации | Средний | UI-предупреждение + маскирование при отображении |
| Refresh-токены не отзываются при деактивации | Низкий | Сессия истекает за 1 час; блокировка входа мгновенная |
| In-memory session store | Низкий (dev) | Задокументировано: замена на Redis/PostgreSQL в production |
| Email собирается, но не используется | Низкий | Оставить как опциональное поле для будущих уведомлений |
