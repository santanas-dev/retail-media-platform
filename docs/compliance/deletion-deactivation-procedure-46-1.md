# Deletion / Deactivation Procedure — 46.1

> Retail Media Platform — процедура удаления, деактивации и анонимизации
> Версия: 1.0 | Дата: 2026-06-29 | Этап: 46.1

## 1. Принцип: логическое удаление

Физическое удаление записей из БД **не применяется** для:
- Пользователей (сохраняется audit trail)
- Кампаний (сохраняются публикации, отчёты)
- Креативов (сохраняется история версий)
- Любых объектов с FK-зависимостями в audit

Вместо этого используется **деактивация** (soft delete).

## 2. Процедура деактивации пользователя

### 2.1. Кто может инициировать
- `system_admin` — полные права
- `security_admin` — управление пользователями

### 2.2. Кто утверждает
- Деактивация не требует отдельного approval в v1
- Все действия аудируются через `admin_audit_events`

### 2.3. Что происходит в БД

```
users.is_archived = true
users.is_active = false
users.archived_at = now()
users.archived_by = actor_user_id
```

### 2.4. Что сохраняется
- `created_by` (кампании, креативы) — остаётся ссылка на archived user
- `approved_by` — остаётся
- `uploaded_by` — остаётся
- Все записи аудита — остаются

### 2.5. Что блокируется
- Вход в систему: `is_archived == True` → 403 "Account is archived"
- Новые действия от имени пользователя

### 2.6. Что НЕ происходит (v1)
- ❌ Автоматический отзыв refresh-токенов (сессия истекает за 1 час)
- ❌ Автоматическая анонимизация display_name в старых объектах

### 2.7. Проверка
```sql
SELECT username, is_active, is_archived, archived_at, archived_by
FROM users WHERE username = '<target>';
```

## 3. Процедура архивации кампании

### 3.1. Действие
- `status = 'archived'` (через стандартный lifecycle)
- Не удаляет связанные креативы
- Не удаляет публикации
- Не удаляет audit trail

### 3.2. UI
- Портал показывает статус «Архивирована»
- Кнопки действий недоступны

## 4. Процедура архивации креатива

### 4.1. Действие
- `status = 'archived'`
- Файлы в MinIO сохраняются (в рамках retention)
- Связанные версии сохраняются

## 5. Безопасность

- **Защита self**: нельзя деактивировать самого себя
- **Защита last admin**: нельзя деактивировать последнего активного system_admin
- **Audit trail**: каждое изменение статуса записывается в `admin_audit_events`

## 6. Roadmap

- Автоматический отзыв refresh-токенов при деактивации
- Автоматическая анонимизация display_name в archived объектах
- Периодическая очистка истёкших refresh_tokens
