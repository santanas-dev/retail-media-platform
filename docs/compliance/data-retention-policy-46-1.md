# Data Retention Policy — 46.1

> Retail Media Platform — политика сроков хранения данных
> Версия: 1.0 | Дата: 2026-06-29 | Этап: 46.1

## 1. Общие принципы

1. **Хранение по назначению** — данные хранятся ровно столько, сколько необходимо для бизнес-процессов и compliance
2. **Логическое удаление** — физическое удаление заменено на деактивацию/архивацию там, где нужен audit trail
3. **Массовая очистка** — только по утверждённому регламенту с предварительным backup
4. **Анонимизация** — персональные данные в бизнес-объектах анонимизируются при архивации

## 2. Сроки хранения по категориям

### 2.1. Пользователи и доступ

| Таблица | Срок | Действие по истечении |
|---|---|---|
| `users` | Весь период + 1 год | Деактивация (is_archived=true) |
| `user_roles` | Весь период + 1 год | Сохраняется для audit trail |
| `user_rls_scopes` | Весь период + 1 год | Деактивация (is_active=false) |
| `refresh_tokens` | 7 дней (срок токена) | Автоматическое удаление истёкших |
| `mfa_settings` | Весь период + 1 год | Сохраняется для audit trail |

### 2.2. Аудит

| Таблица | Срок | Примечание |
|---|---|---|
| `login_audit_events` | 1 год | IP и UA — хеши |
| `admin_audit_events` | 3 года | Действия администрирования |
| Бизнес-аудит (approvals, publications) | 3 года | Операционный audit trail |

### 2.3. Бизнес-данные

| Таблица | Срок | Действие |
|---|---|---|
| `advertisers`, `brands`, `contracts` | Договор + 3 года | Архивация |
| `campaigns` | Кампания + 1 год | Архивация (status=archived) |
| `creatives` + `creative_versions` | Кампания + 1 год | Архивация |
| `approval_requests` | 3 года | Сохраняется для audit |
| `schedules` + `schedule_slots` | Кампания + 1 год | Архивация |
| `publication_batches` | 3 года | Для аудита доставки |

### 2.4. Отчёты и метрики

| Таблица | Срок | Действие |
|---|---|---|
| `campaign_delivery_snapshots` | 1 год | Удаление |
| `kso_proof_of_play_events` | 90 дней | Удаление (сырые события) |
| `proof_of_play_events` (enterprise) | 1 год | Агрегация → удаление сырых |

## 3. Процедура деактивации (вместо удаления)

- **Пользователи**: is_archived=true, is_active=false, archived_by, archived_at
- **Кампании**: status=archived, сохраняется created_by/approved_by
- **Креативы**: status=archived, сохраняется uploaded_by
- **Refresh-токены**: отзыв вручную (в roadmap: авто-отзыв при деактивации)

## 4. Процедура массовой очистки

1. Согласование с администратором
2. pg_dump backup
3. CSV экспорт удаляемых данных
4. Транзакционное удаление по FK-зависимостям
5. Запись в audit о факте очистки

## 5. Исключения

- **Не выполнять** массовую очистку без отдельного approval
- **Не удалять** audit trail физически
- **Не удалять** campaign/creative если есть публикации или audit trail
